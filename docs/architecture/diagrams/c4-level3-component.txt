===================== MEMORY STORE PIPELINE (detail) =====================

  memory_store(content, tags, tier)
         |
         v
  +------------------+
  | store_urgent()   |  (bypasses queue for critical memories)
  +--------+---------+
           |
           v
  +------------------+
  | smart_extractor  |  → 6-category classification:
  | (regex + LLM     |    preference | fact | decision | entity | event | pattern
  |  fallback)       |
  +--------+---------+
           |
           v
  +------------------+
  | check_duplicate  |  → LanceDB ANN search (cos ≥ 0.85 = duplicate)
  | (vector dedup)   |    If duplicate: access_count++, worth_success++,
  +--------+---------+    last_accessed=now, effective_half_life++
           |              RETURN updated (no new insert)
           v (unique)
  +------------------+
  | QualityGate      |  → 4-dimension scoring (equal weight 0.25):
  | .score(tier)     |    confidence + relevance + freshness + info_density
  +--------+---------+    ≥0.5 = admit | 0.3-0.5 = low_quality | <0.3 = discard
           |
           v (admitted)
  +------------------+
  | RecMem.store()   |  → Weibull decay init:
  | + decay_init     |    L1: β=1.5, half_life=3d
  | + LanceDB write  |    L3: β=0.7, half_life=90d
  +------------------+    Dual-write: SQLite + LanceDB (vector)


===================== MEMORY GC PIPELINE (weekly) =====================

  MemoryGC.collect()  [~7 day cycle]
         |
         v
  +------------------+
  | mark_decaying()  |  → Weibull batch decay update (Direction A)
  +--------+---------+    composite_score = wilson×0.6 + freshness×0.25
           |              + reinforcement×0.15
           v
  +------------------+
  | merge_similar()  |  → LanceDB ANN cos ≥ 0.70 = merge candidates
  +--------+---------+    composite_score decides survivor (Direction A+B)
           |
           v
  +------------------+
  | forget()         |  → Remove decayed + merged (LanceDB + SQLite)
  +------------------+

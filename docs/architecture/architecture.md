## 1. System Overview

**Plastic Promise** is an AI behavioral governance system built on **Commitment Engineering** — a paradigm that replaces external constraint enforcement with internalized conventions. It provides a complete operating system for AI agents: memory with decay, principles with inheritance, context with proactive supply, trust-driven permission escalation, and a multi-agent autonomous pipeline.

- **Purpose**: Govern AI agent behavior through shared memory, evolving principles, and dynamic trust — not hard-coded rules
- **Key Capabilities**: 41 MCP tools across 11 domains, multi-agent team (Claude PM + Pi Builder/Fixer/Reviewer), autonomous pipeline with zero-token polling
- **Target Users**: AI coding agents (Claude Code, Trae, Pi), their human operators, and downstream agent teams
- **Success Metrics**: Trust score trajectory, memory pool health (L1/L2/L3 distribution), SCARF trend, CEI composite index

## 4. Communication Patterns

```
┌─────────────────────────────────────────────────────────┐
│                    Communication Flow                     │
│                                                           │
│  User ──→ Claude (PM) ──→ task_enqueue ──→ SQLite WAL    │
│     ◀────────────────────────────── SSE /events          │
│                                                           │
│  Daemon ──→ SQLite poll (zero-token) ──→ spawn Pi         │
│  Pi ──→ MCP tools ──→ Memory/Context/Principles           │
│  Pi ──→ /notify ──→ SSE ──→ Claude notified              │
│                                                           │
│  Inter-Agent: tags on SQLite rows (no message bus needed) │
│  Sync calls: MCP protocol (stdio / SSE)                   │
│  Async events: SSE /events stream + /notify endpoint      │
└─────────────────────────────────────────────────────────┘
```

- **Tag-driven state machine**: `task:pending → accepted → active → done → review → reviewed`
- **Zero-token polling**: Daemon reads SQLite directly, no LLM calls for task detection
- **Bi-directional bridge**: `/notify` POST → SSE `/events` stream → real-time agent notification
- **Atomic claim**: `task_claim` uses SQLite transaction, first-come-first-served

## 6. Memory & State Management

| Memory Type | Implementation | Half-Life | Purpose |
|-------------|---------------|-----------|---------|
| **L1 (Hot)** | SQLite + LanceDB | β=1.5, 3 days | Active session context, recent decisions |
| **L2 (Warm)** | SQLite + LanceDB | Auto-promoted from L1 | Frequently accessed, stable knowledge |
| **L3 (Cold)** | SQLite + LanceDB | β=0.7, 90 days | Archived, rarely accessed |
| **Working Memory** | In-process dict | Session lifetime | Current task state, SCARF baseline |

**State Management:**
- `EntityGraph` (Rust/Python): entity relationships and association feedback
- `TrustStore` (SQLite): persistent trust scores with history
- `PrincipleTracker`: principle adherence quantification
- `Skill session state`: tracked via `skill_session_start/complete` MCP tools

## 7. Error Handling Strategy

| Layer | Mechanism | Trigger | Action |
|-------|----------|---------|--------|
| **L0 Hard Boundary** | `soul_enforcer.pre_check` | Dangerous operation detected | Block + trust decay (-0.05) |
| **L1 Trust Constraint** | `defense(action="get")` | Trust < threshold | Restrict or require approval |
| **L2 Immune Patrol** | `soul_audit` (hourly) | 11-dimension scan | Auto-fix Tier1, report Tier2+ |
| **Task Timeout** | Daemon heartbeat | 5 min no response | Reset to pending, escalate count++ |
| **Graceful Degradation** | `_dm_ok` flag | Subsystem failure | Disable failing subsystem, continue |
| **Disaster Recovery** | `domain(action="rebuild")` | Domain corruption | Rebuild from tags |

## 8. Security Model

- **Trust-Driven Authorization**: 4-tier matrix (autonomous/standard/restricted/readonly) maps trust score to tool permissions
- **Atomic Task Claims**: SQLite transactions prevent double-claiming
- **Input Validation**: All MCP tool inputs validated against JSON Schema
- **Audit Trail**: Every action logged via `audit_run`, principle alignment tracked
- **Rate Limiting**: Hunter guild overreach prevention (trust penalty for claiming above rank)

## 10. Scalability Plan

- **LanceDB**: Disk-based ANN index, scales to millions of vectors without memory pressure
- **SQLite WAL**: Concurrent reads + single writer, sufficient for agent team (<100 concurrent)
- **Zero-Token Polling**: Daemon uses SQL queries, not LLM calls — infinite scalability for task detection
- **RRF Fusion**: Combines ANN + FTS + symbolic rules without expensive re-ranking for most queries
- **Caching**: Embedding cache for frequent queries, principle activation results memoized
